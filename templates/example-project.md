The API gateway v2 migration reached final rollout and shipped on 2026-01-15 as
release 2.0. The original 2.0 roadmap item (request-level rate limiting) was
deferred to 2.1.

**Why:** this is release/roadmap state that the code and git history don't make
obvious, and it changes which version number new work targets.

**How to apply:** when picking a target version or discussing rollout readiness,
treat 2.0 as the shipped milestone and 2.1 as the next free slot. Re-verify the
current version with the project's version tool before relying on this.
